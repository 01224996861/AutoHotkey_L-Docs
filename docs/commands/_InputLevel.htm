#InputLevel [v1.1.06+]

Controls which artificial keyboard and mouse events are ignored by hotkeys and hotstrings.

#InputLevel [, Level]

Parameters

Level An integer between 0 and 100. If omitted, it defaults to 0.

General Remarks

This directive is positional: it affects all hotkeys and hotstrings between it and the next #InputLevel directive. If not specified, hotkeys and hotstrings default to level 0.

If an artificial keyboard or mouse event on an equal or lower level is received, it will not trigger the hotkey or hotstring. Since this type of filtering requires the use of the keyboard or mouse hook, a non-zero input level also forces #UseHook into effect. However, hotkeys at level 0 may be implemented with the reg method, in which case they may be triggered by artificial input from any level (even level 0).

The input level of a hotkey or hotstring is also used as the default send level for any artificial input generated by that hotkey or hotstring. Consequently, a non-zero input level causes any input generated by a hotkey to be ignored by the hotkey itself and any hotkeys of a lower level.

AutoHotkey versions older than v1.1.06 behave as though #InputLevel 0 and SendLevel 0 are in effect.

Related

SendLevel, Hotkeys, Hotstrings

Examples

#InputLevel 1
Numpad0::LButton
#InputLevel 0
; This hotkey can be triggered by both Numpad0 and LButton:
~LButton::Msgbox Clicked
